0013 %% Loading event catalogs
0014 % The catalog class is for loading, plotting and analysing event catalogs.
0015 % For a detailed description, read HELP CATALOG.M.
0016 %
0017 % For the purpose of this exercise we will be using data from Redoubt volcano
0018 % from 2009/03/20 to 2009/03/23. We will use two catalogs:
0019 %
0020 % # The real-time catalog, produced using Antelope.
0021 % # The analyst-reviewed offical AVO catalog, produced using Earthworm,
0022 % XPick and Hypoellipse (and then later converted to an Antelope database).
0023 %
0024 % Both catalog segments are distributed in the "demo" directory.
0025 %
0026 % We will now load the real-time catalog into a catalog object. First,
0027 % because we don't know where GISMO is on your system, we have to construct the
0028 % path to the demo directory based on where CATALOG.M resides:
0029 
0030 dirname = fileparts(which('catalog')); 
0031 dbpath = [dirname,'/demo/avodb200903']; 
0032 cobj = catalog(dbpath, 'antelope', 'snum', datenum(2009,3,20), 'enum', datenum(2009,3,23), 'region', 'Redoubt')
0033 
0034 %% Magnitude-time plot

0053 %% Plots of earthquake counts
0054 % For a quick plot of earthquakes per hour, we create an eventrate object
0055 % and then plot it. Here our binsize is 1/24 days, i.e. 1 hour.
0056 erobj = eventrate(cobj, 1/24)
0057 plot(erobj);
0058 
0059 %% Event rates for overlapping time windows
0060 % Sometimes it is desirable to compute event rate metrics for sliding -
0061 % i.e. overlapping - time windows. This is easily done with the 'stepsize'
0062 % parameter. If omitted, stepsize defaults to the binsize - which is the
0063 % length of the time window. So in the previous example, both binsize and
0064 % stepsize were 1.0 hours. But we can just as easily compute an eventrate
0065 % object for the same catalog object with a binsize of 1 hour, and stepsize
0066 % of just 5 minutes.
0067 erobj2 = eventrate(cobj, 1/24, 'stepsize',  5/1440)
0068 plot(erobj2)
0069 
0070 %% Plots of other event rate metrics
0071 % >> plot(erobj) is actually equivalent to typing:
0072 %
0073 % >> plot(erobj, 'metric', 'counts');
0074 %
0075 % The full list of metrics that can be plotted are:
0076 %%
0077 %
0078 % * counts
0079 % * mean_rate
0080 % * median_rate
0081 % * cum_mag
0082 % * mean_mag
0083 % * median_mag
0084 % * energy
0085 %
0086 % All of these are properties of an eventrate object except for energy,
0087 % which is computed from cum_mag on-the-fly.
0088 %
0089 % Several can be plotted at once in subplots of the same figure using a cell array:
0090 %
0091 % >>plot(erobj2, 'metric', {'mean_rate'; 'median_rate'; 'mean_mag'; 'cum_mag'});
0092 %
0093 % Or they can of course be plotted in separate figure windows:
0094 plot(erobj2, 'metric', 'mean_rate')
0095 plot(erobj2, 'metric', 'median_rate')
0096 plot(erobj2, 'metric', 'mean_mag')
0097 plot(erobj2, 'metric', 'cum_mag')
