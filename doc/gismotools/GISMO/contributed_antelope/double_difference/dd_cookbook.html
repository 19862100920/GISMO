SYNOPSIS

DESCRIPTION 

% HypoDD cross correlation toolbox 
 This toolbox of programs carries out waveform cross-correlation on picked
 P and S phases to generate date suitable for inclusion in the hydoDD
 program. This is a high level toolbox designed to be fast and
 user-friendly. To accomplish this, it requires that a number of other
 toolboxes be in place already. Known dependencies include:
       Antelope toolbox
       waveform toolbox
       correlation toolbox
       epoch2datenum.m

0019 %% Data sources
0020 % This toolbox is hardwired to data from Antelope databases. This includes
0021 % origins, phase arrival and waveform data. All tables should be accessible
0022 % via a single database. If you are combining databases, use the descriptor
0023 % file to bring the relevant tables into a single database. At a minimum
0024 % this database muxt include the following tables:
0025 %       origin
0026 %       event
0027 %       assoc
0028 %       arrival
0029 %       wfdisc

0037 %% Creating an 'SCP' file
0038 % Cross correlation is carried out on waveforms of common station, channel
0039 % and phase pick. Some parameters, such as trace window length and
0040 % filtering, may be adjusted differently for different stations. This is
0041 % accomplished in what we refer to as the SCP file. This simple file
0042 % contains all station/channel/phase combinations that actually appear in
0043 % the given database. In is the input file which controls the subsequent
0044 % cross correlation process. The function DD_MAKE_SCP creates a nominal
0045 % version of this file. So long as the general format is not changed, it is
0046 % designed to be hand edited to account for different processing among
0047 % channels. In the following examples, we are using a database named
0048 % 'dbclust'. Details on DD_MAKE_SCP can be found with HELP DD_MAKE_SCP. One
0049 % feature to note however is the ability to specify a preferred horizontal
0050 % component. Catalog data might be picked on either 'E' of 'N' components.
0051 % DD_MAKE_SCP allows collapses these picks onto a single horizontal channel
0052 % for cross correlation, as specified in the function. This only applies to
0053 % 'E' and 'N' components. Picks on different types of channel, for example
0054 % BNE and HHE are not combined. In most cases concurrent picks on different
0055 % types of data channels will not occur. If they however, DD_MAKE_SCP
0056 % provides a good way to check of this.
