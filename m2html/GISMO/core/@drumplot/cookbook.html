DESCRIPTION 

% drumplot cookbook
 In this example you will learn how to create a waveform object by loading
 a SAC file, extract 1 hour of data from that waveform object, and make
 a helical drum recorder plot using the drumplot class. 
 
 Another feature of drumplot is that it allows detected events, from a
 Catalog object, to be superimposed on the helicorder plot. In the example
 below this Catalog is generated by running an STA/LTA detection algorithm
 on the extracted 1-hour waveform object.

0011 %% Load 1 day of data for REF.EHZ on 2009.03.22
0012 
0013 % here we can either use scnlobject or ChannelTag to describe the
0014 % net-sta-loc-chan we want. waveform understands both.
0015 ctag = ChannelTag('AV.REF.-.EHZ')
0016 
0017 % starttime in MATLAB datenum format
0018 snum = datenum(2009,3,22);
0019 
0020 % endtime (1 day after start)
0021 enum=snum+1;
0022 
0023 % create the datasource object - in this case it is a SAC file
0024 ds = datasource('sac', 'SACDATA/REF.EHZ.2009-03-22T00:00:00.000000Z.sac');
0025 
0026 % create the waveform object with this datasource, ChannelTag, starttime
0027 % and endtime
0028 w=waveform(ds, ctag, snum, enum);
0029 
0030 
0031 %% fill gaps, detrend, band pass filter
0032 
0033 % in case there are gaps in the time series (marked by NaN) we can interpolate a
0034 % meaningful value
0035 w = fillgaps(w, 'interp');
0036 
0037 % detrend the time series - this removes linear drift
0038 w = detrend(w);
0039 
0040 % create a Butterworth bandpass filter from 0.5 to 15 Hz, 2 poles
0041 fobj = filterobject('b', [0.5 15], 2);
0042 
0043 % apply the filter in both directions (acausal) - this is a zero phase
0044 % filter which is helpful because it doesn't disperse different frequency
0045 % components. the caveat is that it can spread energy so arrivals may appear
0046 % slighter earlier than they actually are
0047 w = filtfilt(fobj, w);
0048 
0049 %% plot the waveform object
0050 figure
0051 plot(w)
0052 
0053 %% extract the first 1 hour of data and plot with 5 minutes per line
0054 starttime = get(w,'start');
0055 w2=extract(w, 'time', starttime, starttime + 1/24 )
0056 
0057 % make a drumplot object. mpl means minutes per line and is set here to 5.
0058 h2 = drumplot(w2, 'mpl', 5);
0059 
0060 % plot the drumplot object - many events are visible, this is an earthquake
0061 % swarm
0062 plot(h2)
0063 
0064 %% Run an STA/LTA detector on the waveform object to detect events
0065 % Compute STA/LTA ratio. In this example, trigger "ON" when STA/LTA exceeds
0066 % 3, trigger "OFF" when this drops back below 1.5. If the trigger was on
0067 % for at least 2 seconds, declare it as an event.
0068 % All the events get saved into a Catalog object.
0069 
0070 % set the STA/LTA detector
0071 sta_seconds = 0.7; % STA time window 0.7 seconds
0072 lta_seconds = 7.0; % LTA time window 7 seconds
0073 thresh_on = 3; % Event triggers "ON" with STA/LTA ratio exceeds 3
0074 thresh_off = 1.5; % Event triggers "OFF" when STA/LTA ratio drops below 1.5
0075 minimum_event_duration_seconds = 2.0; % Trigger must be on at least 2 secs
0076 pre_trigger_seconds = 0; % Do not pad before trigger
0077 post_trigger_seconds = 0; % Do not pad after trigger
0078 event_detection_params = [sta_seconds lta_seconds thresh_on thresh_off ...
0079     minimum_event_duration_seconds];
0080 
0081 % run the STA/LTA detector. lta_mode = 'frozen' means the LTA stops
0082 % updating when trigger is "ON".
0083 [cobj,sta,lta,sta_to_lta] = Detection.sta_lta(w2, 'edp', event_detection_params, ...
0084     'lta_mode', 'frozen');
