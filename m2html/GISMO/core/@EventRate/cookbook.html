0046 %% Event rates for overlapping time windows
0047 % Sometimes it is desirable to compute event rate metrics for sliding -
0048 % i.e. overlapping - time windows. This is easily done with the 'stepsize'
0049 % parameter. If omitted, stepsize defaults to the binsize - which is the
0050 % length of the time window. So in the previous example, both binsize
0051 % and stepsize were 1.0 hours. But we can just as easily compute an
0052 % EventRate object for the same EventRate object with a binsize of 1 hour,
0053 % and stepsize of 1 minute. This effectively converts a Catalog into a set
0054 % of continuous metrics, measured every minute, directly comparable to
0055 % 1-minute RSAM.
0056 
0057 eventrateObject = catalogObject.eventrate('binsize', 1/24,  'stepsize', 1/1440);
0058 eventrateObject.plot()
